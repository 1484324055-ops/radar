# 雷达 — 使用手册

## 一句话介绍

雷达是一个基于「梳理—合并—突破」方法论的个人工作台。帮你把脑子里的想法理清楚、排出来、打穿它。

---

## 为什么需要这个工具

你是不是经常有这种感觉：

- 脑子里什么都在想，但不知道先干什么
- 每天忙很多事，但没有突破感
- 想法太多，入口太多，责任太多
- 工作抽离后，不知道自己该干嘛

这不是你懒，也不是没有执行力。真正的问题是：想法太多，没有固定的清空和合并机制，大脑会一直被未整理的信息占满。

雷达就是你的「CEO 操作系统」：先清空，再归类；先合并，再突破；先减法，再执行。

---

## 核心方法论：梳理—合并—突破

这套方法来自生财有术亦仁的分享，适合脑子里长期并行很多线索的人（CEO、创业者、主理人）。

### 第一步：梳理（清空脑子）

把脑子里所有未闭环的事情倒出来。不分类、不判断，先倒干净。

- 我最近一直在想什么？
- 哪些事让我烦？
- 哪些项目有卡点？
- 哪些机会值得研究？
- 哪些事其实该停掉？

### 第二步：合并（翻译成行动）

把散乱的想法翻译成可处理的对象：

- 每个想法到底是一个情绪、一个判断、一个项目、还是一个任务？
- 5 个碎片问题可能本质是同一个杠杆点
- 找到「一个动作打穿多件事」的合并点

### 第三步：突破（打穿主战役）

- 每周只定 1-2 个主战役
- 48 小时内必须推进
- 一天结束时，看的是「这个问题有没有被打穿一层」，不是「做了多少事」

## 怎么用雷达

### 安装到手机

1. iPhone Safari 打开 https://radar-six-dusky.vercel.app
2. 点底部「分享」按钮（方框+箭头图标）
3. 往下滑找到「添加到主屏幕」
4. 确认添加 → 桌面出现「雷达」图标
5. 以后从桌面图标打开，全屏运行

### 首次打开

第一次打开雷达会看到 5 步引导：

1. 欢迎介绍
2. 倾倒体验（让你倒第一条想法）
3. 翻译机制
4. 主战役概念
5. 数据回顾

引导过程中可以跳过，也可以跟着走一遍。

---

## 六个页面详解

### 1. 仪表盘（首页）

打开雷达第一眼看到的就是仪表盘。它回答一个问题：「我现在的情况怎么样？」

**今日节奏**
告诉你今天是周几、建议做什么。雷达内置了一周节奏模板：

| 周几 | 建议 | 做什么 |
|------|------|--------|
| 周一 | 梳理日 | 清空脑子，确定本周主线 |
| 周二 | 突破日 | 打穿本周最重要事项 |
| 周三 | 关系日 | 处理人和协作 |
| 周四 | 突破日 | 第二次深度推进 |
| 周五 | 清债日 | 清理悬挂、回复、补资料 |
| 周六日 | 休息日 | 家庭 / 恢复 |

**四维统计卡片**
- 已完成：本周完成的任务数
- 进行中：正在推进的任务数
- 暂停：暂时搁置的任务数
- 待处理：倾倒了但还没翻译的想法数

**本周主战役**
显示你本周设定的主战役和推进计划。如果没有设定，会提示你去设定。

**进行中的事**
最多显示 3 条正在进行的任务，提醒你不要同时开太多。

---

### 2. 倾倒（快速输入）

这是雷达的核心入口。回答一个问题：「脑子里现在在想什么？」

**怎么用**
1. 打开倾倒页面
2. 在大输入框里直接写下来
3. 点「存入待处理」
4. 继写下一条

**关键原则**
- 不分类、不判断、不排序
- 有多少倒多少，倒干净为止
- 可以是任务、焦虑、灵感、待办、想法、人名、任何东西

**语音输入**
点麦克风按钮可以语音输入（需要网络连接）。说完点停止，文字会自动填入输入框。

注意：从主屏幕图标打开时，语音功能可能不可用。如果需要语音，建议用 Safari 打开。

**最近倾倒**
页面底部显示最近 5 条未处理的倾倒，让你知道倒了什么。

---

### 3. 待处理池（翻译区）

回答一个问题：「这些想法需要我做什么？」

倾倒的内容会自动进入待处理池。你需要逐条处理：

**三种处理方式**
1. **变成任务**：点击后选优先级和类别，存入任务池
2. **不需要**：标记为已处理，但不创建任务（可能是情绪或判断，不需要行动）
3. **删除**：彻底移除

**变成任务时的选择**

优先级：
- 主战役：本周最重要的事
- 重要：需要推进但不是主战役
- 普通：一般事项

类别：
- 突破：战略、产品、内容、需要深度思考的事
- 清债：文件、财务、回复、积压的琐事
- 关系：团队沟通、合作伙伴、人脉维护

**已处理区**
页面底部显示已处理的条目，划线表示已关闭。

---

### 4. 任务池（执行看板）

回答一个问题：「我现在手里有多少事？各是什么状态？」

**五个状态**
- 未开始（○）：还没动手
- 进行中（◉）：正在推进
- 已完成（●）：搞定了
- 暂停（⏸）：暂时搁置
- 不做（✕）：战略性放弃

**切换状态**
点任务左侧的图标，状态会循环：未开始 → 进行中 → 已完成 → 暂停 → 未开始...

**展开详情**
点任务卡片展开，可以看到：
- 推进记录：记录每一步进展
- 分配到天：把任务安排到周一~周日
- 删除任务

**筛选器**
顶部有筛选标签：全部 / 未开始 / 进行中 / 已完成 / 暂停

**不做池**
右上角有「不做」入口，显示战略性放弃的任务数量。点进去可以恢复或彻底删除。

---

### 5. 本周（主战役 + 节奏表）

回答一个问题：「这周要打穿什么？」

**主战役区域**
- 点 + 号设定本周主战役
- 写下要打穿什么 + 48 小时推进计划
- 可以记录推进进展
- 每周设 1-2 个，不要贪多

**七天节奏表**
显示周一到周日，每天：
- 建议做什么（梳理/突破/关系/清债/休息）
- 分配到这天的任务列表
- 今天的行会高亮显示

---

### 6. 数据（统计分析）

回答一个问题：「我的效率怎么样？趋势如何？」

**本周概览**
- 本周完成数
- 本周任务总数
- 完成率百分比

**时间分配饼图**
显示本周任务在突破/清债/关系三类上的分布。

关键洞察：
- 如果清债占比太高，说明你一直在处理琐事，需要更多突破时间
- 如果关系占比太高，说明你被会议和沟通占满了
- 理想状态是突破占比最高

**周趋势柱状图**
过去 8 周的任务数和完成数对比。看趋势比看单周更重要。

**主战役历史**
过去 4 周的主战役记录，回顾你每周在打穿什么。

---

### 7. 设置

**主题切换**
三套主题可选：
- 浅色（默认）：白色背景，适合白天
- 深色：深灰背景，适合夜间
- 暖光：暖白背景，适合喜欢暖色调

**数据管理**
- 导出数据：把所有数据导出为 JSON 文件，建议定期备份
- 导入数据：从 JSON 文件恢复数据
- 清除数据：清空所有数据（不可恢复）

## 全局功能

### 快速倾倒按钮

除了倾倒页面，右下角有一个紫色 + 号悬浮按钮。在任何页面都能快速倾倒想法，不用切换页面。

点开后是一个弹出面板：输入想法 → 存入 → 自动进入待处理池。

## 推荐工作节奏

### 周一：梳理日

1. 打开雷达
2. 去倾倒页面，把脑子里所有东西倒出来（10-20 分钟）
3. 去待处理池，逐条翻译成任务
4. 去本周页面，设定 1-2 个主战役 + 48 小时推进计划
5. 把任务分配到这一周的各天

### 周二/周四：突破日

1. 打开仪表盘，看今日节奏确认是突破日
2. 集中精力推进主战役
3. 完成后在任务里记录推进进展
4. 一天结束时，看主战役有没有被打穿一层

### 周三：关系日

1. 处理团队沟通、合作伙伴、人脉维护
2. 这类事不适合夹在深度工作中间做

### 周五：清债日

1. 清理积压的待处理池
2. 回复未回的消息
3. 整理文件、补资料
4. 下午做收口复盘：看本周完成情况，定下周预备事项

### 每天：随时倾倒

任何时候有想法，点右下角 + 号，倒出来就走。不用管分类，周一再统一处理。

## 五个池子

| 池子 | 在哪里 | 放什么 |
|------|--------|--------|
| 倾倒池 | 倾倒页面 | 所有未加工的想法、焦虑、灵感 |
| 待处理池 | 待处理页面 | 已倾倒但还没翻译成任务的条目 |
| 任务池 | 任务页面 | 已翻译、有明确状态的任务 |
| 主战役 | 本周页面 | 本周最重要的 1-2 件事 |
| 不做池 | 任务页面→不做 | 战略性放弃的事 |

## 数据安全

- 数据存在你手机浏览器的 localStorage 里
- 不上传到任何服务器
- 清除浏览器缓存会丢失数据
- 建议每周导出一次 JSON 备份

**导出方法**：设置 → 导出数据 → 保存文件

**导入方法**：设置 → 导入数据 → 选择之前导出的 JSON 文件

---

## 常见问题

**Q: 数据会丢吗？**
A: 如果你清除浏览器缓存或卸载浏览器，数据会丢。建议定期在设置里导出备份。

**Q: 能多设备同步吗？**
A: 目前不支持。数据存在单台设备上。可以通过导出/导入在设备间迁移。

**Q: 语音不能用？**
A: 从主屏幕图标打开时，iOS 的语音识别可能不可用。用 Safari 打开可以正常使用语音。

**Q: 离线能用吗？**
A: 可以。雷达配了 Service Worker，加载过一次后，没网也能正常使用。

**Q: 怎么更新？**
A: 代码更新后自动部署。手机端刷新页面就拿到最新版。

## 你的最佳工作原则

你不是每天平均产出型，而是周期性清空、合并、突破型。

不要强迫自己每天都像机器一样线性推进。你更适合：

- 一周一次大梳理
- 二到三个深度执行块
- 若干关系处理块
- 再加一次收口复盘

你真正要保护的不是时间，而是三种状态：

1. **清空状态**：允许自己把混乱全部倒出来
2. **判断状态**：从一堆事项里找出最重要的合并点
3. **突破状态**：连续几个小时只推进一个核心问题

只要这三个状态被保护住，你的效率会比每天排满 todo 高很多。

---

## 最后

梳理日必须有一个硬约束：每次梳理结束，必须选出一个 48 小时内要推进的突破点。

否则梳理会变成高级内耗。

梳理完不能只是「最近要重构 XX 系统」，而要变成「48 小时内完成：列出 5 类信息、定义筛选规则、做最小版模板、找一条真实消息流测试」。

梳理日必须连接执行日。

雷达帮你做到这一点。
